# DrillSwipe product brief

## 対象パターン

縦スクロールの短尺フィード、1カード1テーマ、保存、復習キュー、プロフィール、音声/字幕の体験パターンを抽象化する。実在サービスの名称、ロゴ、コピー、配色は使わない。

## 差別化したゴール

DrillSwipe は娯楽フィードではなく、15秒前後の学習ドリルを反復するサンプル。学習者は字幕付きの短尺プレースホルダーを見ながら即答し、弱い項目を保存または復習キューへ戻す。

## 非ゴール

- 実動画の配信、投稿、コメント、フォロー関係の実装。
- 実決済、実認証、外部アカウント連携。
- 実在サービスのロゴ、商標、UI文言、配色の再現。

## 主要ユーザーフロー

1. ホームで現在のドリル、進捗、状態、復習キューを確認する。
2. ドリル詳細で字幕と問題を読み、選択肢から正解/不正解を確認する。
3. ドリルを保存し、復習キューへ追加/解除する。
4. 次のドリルへ縦型フィードのように遷移する。
5. プロフィールで正解数、保存数、復習数、視聴履歴を確認し、履歴を削除する。

## 失敗状態

- `mock-api`: online / offline / timeout
- `mock-auth`: anonymous / premium
- `mock-billing`: ok / failed
- `mock-media`: ok / failed

UIはこれらを表示し、E2Eは `/__control/state` から状態を切り替えて確認する。
