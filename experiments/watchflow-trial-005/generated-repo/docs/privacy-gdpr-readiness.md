# GDPR準備メモ

WatchFlow はローカルモック実装ですが、公開サービス化を想定して以下の論点を先に設計メモとして残します。

| 項目 | 設計メモ |
| --- | --- |
| 閲覧履歴 | 保存目的、保存場所、停止設定、ユーザー削除導線を明示する。匿名利用時は端末内保存を既定にし、同期には同意を求める。 |
| コメント | 投稿者ID、本文、通報状態、モデレーション履歴を分離する。削除要求時は本文削除と監査ログ保持の境界を決める。 |
| 課金状態 | 決済情報そのものは保持せず、決済事業者の参照ID、プラン、支払い失敗状態、監査ログのみを最小化する。 |
| Cookie/storage | 必須、分析、パーソナライズを分類する。同意前に非必須CookieやlocalStorageを書き込まない。 |
| 同意 | 同意文面バージョン、取得時刻、撤回時刻、対象カテゴリを保存する。撤回後は非必須処理を停止する。 |
| 削除要求 | 本人確認、受付、実行、完了通知、例外保持の理由をワークフロー化する。 |
| データエクスポート | 閲覧履歴、コメント、課金ステータス、同意履歴をJSONまたはCSVで出力できるようにする。 |
| 保持期間 | 閲覧履歴90日、コメントは公開中または削除要求まで、課金監査ログは法定期間など目的別に設定する。 |

## 実装時の注意

- UI文言は `lib/i18n/` に集約し、同意文面のバージョン管理を可能にする。
- API client と mock adapter の境界を維持し、将来の実API接続時にPIIの流れをレビューしやすくする。
- 削除要求とデータエクスポートは管理者機能ではなく、ユーザー本人が開始できる導線にする。
